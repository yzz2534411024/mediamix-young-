# MediaMix - 跨平台媒体聚合应用

一款基于 Flutter 开发的跨平台视频聚合播放应用，支持多源视频采集、在线播放、下载管理等功能，内置全面的播放性能优化体系。

## 功能特性

### 核心功能
- **多源视频聚合** — 支持添加多个 CMS 视频源，自动聚合搜索结果
- **在线播放** — 基于 media_kit 的高性能视频播放器，支持硬解码
- **换源播放** — 播放中一键切换不同源的同名视频
- **下载管理** — 支持视频离线下载与后台管理
- **播放记忆** — 自动保存播放进度，下次继续观看
- **直播电视** — 支持 IPTV 直播源导入与播放

### 播放器优化（v1.0 新增）
- **接口解析优化** — DNS 预解析、HTTPDNS、接口预请求与并行化、CDN 调度、指数退避重试
- **视频解码优化** — 硬解码优先 + 自动降级、解码器配置优化
- **播放流畅性** — ABR 自适应码率、动态缓冲水位线、Seek 关键帧优化、音视频同步监控
- **加载缓存** — 四级缓存架构（L1帧缓存/L2码流/L3完整视频/L4分片）、LRU+优先级淘汰、智能预加载
- **倍速播放** — 0.25x 步进、高倍速优化
- **画中画** — 支持 PiP 模式，后台继续播放音频
- **功耗管理** — 三档功耗模式（全性能/均衡/省电），电池电量自动切换
- **性能监控** — 11 个埋点事件、QoE/QoS 双维度指标、实时告警

## 快速开始

### 环境要求

- Flutter SDK >= 3.0.0
- Dart SDK >= 3.0.0
- Android Studio / VS Code
- Android SDK (API 21+)
- iOS: Xcode 15+ (如需 iOS 构建)

### 安装与运行

```bash
# 1. 克隆项目
git clone <repository-url>
cd mediamix

# 2. 获取依赖
flutter pub get

# 3. 生成数据库代码（如修改了 database.dart）
flutter pub run build_runner build --delete-conflicting-outputs

# 4. 运行应用
flutter run

# 5. 构建 APK
flutter build apk --release
```

### 生成应用图标

如需更换应用图标，替换 `assets/icons/icon.png` 后运行：

```bash
dart run flutter_launcher_icons
```

## 播放器优化架构

### 接口解析优化链路

```
用户点击 → DNS预解析(省100-200ms) → 接口预请求(省300-500ms)
         → 并行换源请求 → CDN调度 → 开始播放
```

### 四级缓存架构

```
┌─ L1: 内存帧缓存 (纳秒级, 20条目) ─────────────┐
├─ L2: 内存码流缓存 (纳秒级, 50条目) ────────────┤
├─ L3: 磁盘完整视频缓存 (毫秒级, ≤2GB) ──────────┤
└─ L4: 磁盘分片缓存 (毫秒级, HLS/DASH segments) ─┘
```

### ABR 自适应码率

| 条件 | 动作 |
|------|------|
| 缓冲 < 5s | 立即降级画质 |
| 缓冲 > 30s 且带宽充足 5s | 升级画质 |
| 切换间隔 < 10s | 不切换 |

### 功耗模式

| 模式 | 帧率 | 分辨率 | 预加载 | 弹幕 |
|------|------|--------|--------|------|
| 全性能 | 60fps | 1080p | 开启 | 开启 |
| 均衡 | 30fps | 720p | 开启 | 开启 |
| 省电 | 24fps | 480p | 关闭 | 关闭 |

## 长期优化与维护方案

### 阶段一：稳定性加固（1-2 个月）

- **缓存一致性保障** — 增加磁盘索引校验机制，启动时检测并修复 L3/L4 缓存索引与实际文件的不一致；增加缓存写入原子性保护，避免异常退出导致索引损坏
- **网络容错增强** — DNS 预解析增加多 DNS 服务器轮询（系统 DNS / 114.114.114.114 / 8.8.8.8），单点失败自动切换；CDN 调度增加 UDP 探测补充 TCP 测速，提升弱网场景下的节点选择准确性
- **播放异常兜底** — 增加播放器状态机异常检测，对死锁、黑屏、无声等异常状态自动恢复；完善错误重试策略，区分网络错误、解码错误、源站错误并分别处理
- **数据上报可靠性** — 增加上报数据本地持久化队列，确保 WiFi 切换或应用重启后不丢失待上报数据；增加上报去重机制，避免重复提交

### 阶段二：性能深度优化（2-4 个月）

- **缓存策略智能化** — 基于用户观看习惯（观看时段、偏好类型、重播频率）动态调整缓存策略；引入贝叶斯预测模型预判用户下一步操作，提前预热缓存
- **ABR 算法升级** — 从基于缓冲水位的简单策略升级为基于机器学习的吞吐量预测模型，结合历史带宽数据和实时网络探测，更精准地选择码率档位
- **预加载精细化** — 根据用户停留时长和跳出率数据，动态调整预加载深度；增加预加载取消回收机制，当用户离开播放页时及时释放已预加载资源
- **内存优化** — L1/L2 缓存增加内存压力感知，在系统内存紧张时自动缩减缓存容量；增加帧缓存生命周期管理，避免长期驻留导致内存泄漏
- **启动加速** — 服务初始化改为懒加载模式，按需启动而非全部预初始化；DNS 预解析与缓存索引加载并行化，减少启动阻塞时间

### 阶段三：体验提升（4-6 个月）

- **离线模式增强** — 支持 WiFi 环境下自动缓存用户收藏和追剧列表中的视频，离线时提供完整观看体验；增加缓存内容管理和过期提醒
- **多设备同步** — 增加播放进度、收藏列表的云端同步能力，支持多设备间无缝切换
- **无障碍与国际化** — 字幕服务增加 ASS/SSA 样式渲染支持；增加 i18n 框架，支持多语言界面切换
- **画中画增强** — 支持画中画模式下显示字幕叠加；增加迷你播放器模式，可在应用内其他页面悬浮播放

### 阶段四：架构演进（6+ 个月）

- **插件化架构** — 视频源解析器插件化，支持社区贡献第三方源解析规则；缓存策略和预加载策略可配置化，支持用户自定义
- **监控体系完善** — 增加服务端指标聚合看板，实时展示全量用户的 QoE/QoS 分布；增加异常自动归因，将卡顿/首屏慢等问题自动关联到网络/解码/缓存等根因
- **跨平台统一** — 统一 Android/iOS/Desktop 的原生通道实现，消除平台差异；功耗管理增加 iOS 低电量模式适配和桌面端 GPU 调度
- **测试覆盖** — 建立核心服务的单元测试和集成测试基线，目标覆盖率 ≥ 70%；增加 CI 自动化性能回归检测，防止优化指标退化

### 持续维护事项

| 事项 | 频率 | 说明 |
|------|------|------|
| 缓存容量巡检 | 每周 | 检查 L3/L4 缓存是否超出磁盘配额，清理过期和低优先级缓存 |
| 依赖版本更新 | 每月 | 更新 Flutter/Dart 及第三方依赖，关注安全补丁和 breaking changes |
| 性能指标回顾 | 每月 | 分析 QoE/QoS 趋势，识别退化指标并制定优化措施 |
| 数据库迁移测试 | 每次发版 | 验证 schema 升级迁移路径的正确性和数据完整性 |
| 隐私合规审查 | 每季度 | 审查数据采集范围和上报内容，确保符合最新隐私法规要求 |
| 缓存淘汰策略调优 | 每季度 | 根据命中率数据调整 LRU 参数、淘汰阈值和保留策略 |
