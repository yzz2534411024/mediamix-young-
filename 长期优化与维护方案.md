# Young 开发维护备忘

> 最后更新：2026-06-25
> 定位：个人/少数人使用的视频聚合播放工具
> 代码规模：lib/ 50 个 Dart 文件，约 15K 行手写代码，25 个测试文件

---

## 1. 已完成的事

### 播放核心
- 四级缓存架构（L1 内存 / L2 分段 / L3 完整文件 / L4 预加载）+ LRU 淘汰
- 边播边缓存代理（LocalProxyServer，Range 支持，512KB 分片）
- 播放器缓存集成（PlayerPage 查 getCachePath，清晰度降级命中）
- ABR 控制器 + 动态缓冲水位线
- 音视频同步三级纠正（微调速率 / Seek / 跳帧）
- 弱网重连（指数退避 1s~32s + 网络恢复自动续播）
- 降级策略（同 URL 重试 → 切清晰度 → 错误对话框）
- 清晰度切换（顶栏按钮、跨源 URL 收集、保持播放位置）
- 快进间隔选择（长按选 5/10/30/60s，双击左右半屏 ±10s）
- PiP 降分辨率（进入切最低清晰度，退出恢复）
- 后台播放降频（AV 同步 2s→10s，进度保存 10s→30s）

### 解码与设备
- 硬解码探测（H.264/H.265/VP9/AV1 × 分辨率）+ 失败自动降级软解
- 功耗三档模式（电池检测 + 自动切换）
- 字幕服务（二分查找、多轨道、LRU 缓存、外部 URL）

### 网络与源
- DNS 预解析 + HTTPDNS（Google/Cloudflare，fallback 系统 DNS）
- 接口预请求 + 并行换源 + 指数退避重试
- CMS 源维护：移除 5 个失效源、修复 3 个 URL、新增 11 个，共 15 个可用源
- Spider 模块：cms_spider / json_spider / xpath_spider / spider_registry

### 架构
- PlayerPage 从 2367 行拆分为 UI 层(1345行) + PlayerCoreManager(763行) + 三个引擎
- 引擎接口：CacheEngine / PlaybackErrorHandler / MetricsEngine
- 数据库 AppDatabase 单例模式，消除 SQLite 锁冲突
- 代码去重：BandwidthEstimator / Semaphore / Hash 算法 / L1-L2 淘汰

### 测试
- 25 个测试文件，约 280+ 测试用例，覆盖核心播放链路
- 覆盖模块：VideoCacheService、BufferManager、ABRController、PreloadService、
  PlayerMetricsService、SubtitleService、DeviceCapabilityService、NetworkEngine、
  PlayerCoreManager、PlaybackErrorHandler、MetricsEngine、CacheResolveResult、HashUtils

## 4. 不做的事

以下功能对个人/少数人使用的项目投入产出比不合理，明确不做：

| 不做 | 理由 |
|------|------|
| P2P 分发网络 | 需要 Tracker 服务器运维 + WebRTC 信令 + NAT 穿透，没有 CDN 成本问题 |
| 端侧 AI 超分辨率 | ONNX Runtime + NPU 适配 + 模型调优，工作量巨大 |
| 端侧 AI 智能预加载 | 联邦学习/决策树训练，无意义 |
| 跨平台统一渲染 | Impeller 已经在做，不需要自研 |
| 弹幕独立渲染层 | 不是弹幕视频站 |
| A/B 实验框架 | 没有用户分流需求 |
| 灰度发布 | 不需要 |
| 崩溃监控（Crashlytics） | 自己用，崩了自己知道 |
| 数据上报/监控大盘 | 上报给谁？看 log 够了 |
| 性能回归检测矩阵 | 过度工程 |
| 无障碍/适老化 | 无合规要求 |
| 进一步架构拆分（引擎可替换） | 当前架构够用，不需要给谁替换 |
| HLS/DASH 多码率 | 第三方 CMS 源无法配合 |
| 解码器复用池 | media_kit 底层已管理，Dart 层做收益有限 |
| GOP 对齐 / Seek 关键帧索引 | 依赖服务端改造，无控制权 |

## 5. 维护

### 自动化（一次配好，不管）
- GitHub Actions：push 时跑 `flutter analyze` + `flutter test`
- 本地打包：`flutter build apk --release`

### 手动（遇到问题时做）
- `flutter pub outdated` — 升级依赖前看一眼
- 内存泄漏 — 长时播放感觉卡了就 DevTools 查一下
- media_kit 版本更新 — 关注 changelog，稳定后再升

### 约束
- 新文件尽量不超过 300 行
- 新功能先手动测一遍再写测试
- 改核心逻辑后跑一遍 `flutter test`
